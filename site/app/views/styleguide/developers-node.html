Node.js on OpenShift

Overview

With Openshift you upload your code using Git and OpenShift configures, secures and scales the web, application and database tiers so you don't have to.

Your application is hosted on "gears" which are configured with a set amount of memory and storage. You define your application's stack using "cartridges", which are frameworks and databases. For example: Node.js is an application type and MySQL is a cartridge. When you sign up you get access to three gears free for a total of 1.5 GB of RAM and 3 GB of storage.

Let's Get in the Cloud

There are 2 options to deploy your Node.js application to the cloud — OpenShift Management Console (easiest) and the OpenShift (RHC) command line tools

Using the OpenShift Management Console

Using the OpenShift Management Console is going to be the fastest and easiest way to get a basic Node.js application deployed. If you are going to eventually deploy your own code or want access to more functionality than the Management Console offers, we recommend getting started with the rhc command line tools.

  1. Sign up for an OpenShift account — an email is all you need to provide. Once you have verified your account and accepted the legal terms, you'll automatically be taken to the OpenShift Management Console create an application process.

  2. In the "Web Cartridges" section select Node.js 0.6. Next, give your new application a name. If this is your first application you will need to create a namespace for all your applications to be grouped under. This step will define the public URL for your app.

  3. Congratulations, your application is up and ready! It really is that easy. You've now got a basic Node.js app running in the cloud courtesy of OpenShift.

If you want to make some changes to the default app, upload your own code, add cartridges like MySQL or simply learn more about hosting Node.js applications on OpenShift, check out the Developer Center.

Using the Command Line

The rhc CLI is for developers who prefer the terminal and want a fully featured environment for managing their OpenShift applications.

  1. Sign up for an OpenShift account — an email is all you need to provide. Once you have verified your account and accepted the legal terms, you'll automatically be taken to the OpenShift Management Console. We are going to use the command line tools, so let's open up a terminal instead.

  2. Install the rhc client tools
    Getting the rhc client tools installed takes just one command. Just follow the instructions on the Getting Started page. You will need to have Git installed and depending on whether you use Windows, Mac or Linux, it's going to be some variant of "gem install rhc."

  3. Create your first Node.js application

    Now that you have the tools installed, execute

    $ rhc setup

    This command sets up your SSH key so that you can securely communicate with Git and creates a namespace for all your applications to be grouped under.

    4. Next, execute the following command:

    $ rhc app create -a  -t Node.js-0.6

    This will give your new app a name and define the public URL for your app.

  4. That's it! You've now got a basic Node.js app running in the cloud courtesy of OpenShift. Not as easy as using the Management Console, but pretty darn close.

If you want to make some changes to the default app, upload your own code, add cartridges like MySQL or simply learn more about hosting Node.js applications on OpenShift, check out the Developer Center.
